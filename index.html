scalafix - a Scala rewrite tool


0.1.0

Scalafix is a new tool to help Scala 2.x compile with Dotty, a next generation compiler for Scala. This effort follows the Scala Center Advisory Board proposal: Clarification of Scala to Dotty migration path.

Read the announcement of the 0.1.0 release here.

The project is still in early days of development, so there is not much to see here yet. Currently, work on scalafix happens mostly in scala.meta, the metaprogramming toolkit that powers the scalafix Rewrites.

The name "scalafix" is inspired by gofix, a similar tool used to migrate go code.

Installation


Command line

There are several ways to install the command line interface.

scalafix.jar

You can download a pre-compiled jar from the releases page. Unzip the tar and run bin/scalafix.

Build from source

  1. Clone the repo.
  2. Run sbt cli/assembly, master branch should pass tests.
  3. Run bin/scalafix.

--help

Once installed, please refer to the scalafix --help page for usage instructions. 
scalafix 0.1.0
Usage: scalafix [options]
  --usage  
        Print usage and exit
  --help | -h  
        Print help message and exit
  --rewrites  <value>
        Rules to run, one of: ProcedureSyntax, VolatileLazyVal
  --in-place | -i  
        If true, writes changes to files instead of printing to stdout.
  --parallel  
        If true, uses all available CPUs. If false, runs in single thread.
  --debug  
        If true, prints out debug information.

Nailgun

Scalafix has support for nailgun, but the setup is quite involved:
  1. Install the nailgun client. On OSX, this can be done with homebrew install nailgun. Make sure that ng is in your PATH.
  2. Clone the repo.
  3. sbt cli/pack
  4. cd cli/target/pack
  5. make install (adds scalafix_server_ng to $PATH)
  6. Symlink the scalafix_ng script to your $PATH. Feel free to name the script scalafix instead of scalafix_ng. The script will automatically start the nailgun server if the server is turned off.

SBT

addSbtPlugin("ch.epfl.scala" % "sbt-scalafix" % "0.1.0")
  • The command sbt scalafix runs all available Rewrites.

Rewrites


Scalafix is a very new tool, more rewrite rules will come in future releases.

ProcedureSyntax

"Procedure syntax" is not supported in Dotty. Methods that use procedure syntax should use regular method syntax instead. For example, 
// before: procedure syntax
def main(args: Seq[String) {
  println("Hello world!")
}
// after: regular syntax
def main(args: Seq[String): Unit = {
  println("Hello world!")
}

VolatileLazyVal

Adds a @volatile annotation to lazy vals. The @volatile annotation is needed to maintain thread-safe behaviour of lazy vals in Dotty. 

// before
lazy val x = ...
// after
@volatile lazy val x = ...

With @volatile, Dotty uses a deadlock free scheme that is comparable-if-not-faster than the scheme used in scalac.
